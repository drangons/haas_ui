Facelet Title
The Sahara plugin provision Hadoop on Openstack.
What is Hadoop?
To simplify cluster provisioning the concept of templates are used. There are two kinds of templates: Node Template and Cluster Template. Templates has parameters that the user can set. This template is used to launch and configure the hadoop cluster. The templates can be reused for launching the clusters.
Currently two version of hadoop are supported. v1.2.1 and v2.3.0
What is a Node Template?.
Node templates are the means by which the user can specifies which hadoop process are running in the cluster.
In the hadoop v1.2.1 user can specify NameNode,DataNode,SecondaryNameNode,Oozie, TaskTracker,JobTracker,HiveServer
In Hadoop v2.3.0 NameNode,DataNode,ResourceManager,NodeManager,HistoryServer.
What is a Cluster Template?
Cluster template specifies the number of nodes in the cluster. The cluster template specified should be consistent.
For Hadoop version 1.x.x Cluster must contain exactly one namenode Cluster can contain at most one jobtracker Cluster can contain at most one oozie and this process is also required for EDP Cluster can’t contain oozie without jobtraker Cluster can’t have tasktracker nodes if it doesn’t have jobtracker
For Hadoop version 2.x.x Cluster must contain exactly one namenode Cluster can contain at most one resourcemanager Cluster can contain at most one historyserver Cluster can contain at most one oozie and this process is also required for EDP Cluster can’t contain oozie without resourcemanager and without historyserver Cluster can’t have nodemanager nodes if it doesn’t have resourcemanager
Note
1. The user has to specify the node template and cluster template and start the cluster.
2.To access the HDFS and Oozie through web interface the ports 50070 and 50030 should be opened in security rules.
3. The number of new floating Ips that can be created should be equal or more than the number of the nodes in the cluster. THe floating Ips will not be reused and hence the user has to delete it after the cluster is deleted.
